## 自己PR
私は、仕事において以下の3つを大事にしています。

### 組織全体の生産性を高めること
私は、組織全体の生産性を向上させるための取り組みを重視しています。具体的には、短時間で質の高いアウトプットを実現する仕組みを組織内に作ることに力を入れています。

短時間で質の高いアウトプットを出す組織にするため、特に重要なのは以下の2点です。

* 組織内で情報を容易に取得できること
* 個人のタスクと組織の目的が明確であること

エンジニアリング組織においてアウトプットの源泉は情報です。組織内の情報を素早く取得できることは、生産性の向上に直結します。このために、

* 既知の情報を集約して、絶えずメンテナンスする
* 他者が持つ情報を取得しやすいコミュニケーションフローを設計する
* コミュニケーションを円滑に行うために心理的安全性を確保する

ことを実現する仕組みを整えています。

また、個人のタスクが明確であればリソース効率・フロー効率が向上します。タスク着手までのリードタイムが短縮され、作業者自身がゴールを探る必要がなくなるからです。

さらに、組織の目的が明確であればタスクが作成された理由が理解できるため、各人が実行時に迷うことがなくなります。

このため、個々が自律的に自分がすべきタスクを認識できる仕組みを構築し、タスクの位置付けを明確にすることを心がけています。

### 学び続けること
私は、公私問わず日々の生活での意思決定は学びによって支えられていると考えています。納得のいく決断を下すためには多角的に検討する必要があり、そのためには幅広い学びが不可欠です。

学習の基盤として読書を重視しており、学術研究がベースになっている書籍を選んでより確からしい情報を得るよう努めています。読書は全体像の把握から始め、必要な箇所を拾い上げて精読するスタイルを取っています。この読書法は「社外活動」の項に書いた[Qiita記事](https://qiita.com/expajp/items/37d799d287c3fff66cf5) にもまとめています。

最近では「人がいかにして学習するか」にも興味を持って認知科学の書籍を読み、自分なりに体系化しました。そしてその学びをもとに、[Developers Summit 2024 Summerにて発表](https://speakerdeck.com/expajp/how-do-engineering-managers-continue-to-learn-and-grow)を行いました。

なお、実際に読んだ本は[ブクログの本棚ページ](https://booklog.jp/users/expajp) でご確認いただけます。

### 科学的であること
ソフトウェアプロダクトの運用において重要なのは仮説検証型のアプローチであり、これを堅持するには科学的であることが重要だと考えています。

科学的であるというのは、次のことを指します。

* 反証可能な仮説を立てること
* 仮説の尤もらしさは証拠（測定データ・実際の出来事）に基づいて判断すること
* すべての仮説は仮説に過ぎず、未来永劫正しいとは保証できないこと

この姿勢を貫くことで、冷静で公正な判断が可能になります。つまり、明確な基準に基づき証拠をもとに判断することで、全員が納得する結論を導き出せます。また、正しい可能性が低い説は、関係性に配慮しつつも却下することができます。

この姿勢は、組織内での信頼関係を築く上で重要であると考えています。
